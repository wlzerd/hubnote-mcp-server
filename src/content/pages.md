# 페이지

hubNote 의 모든 콘텐츠는 페이지로 표현됩니다. 일반 노트, 데이터베이스, HTML View 모두 페이지의 종류입니다.

## 생성

좌측 사이드바의 + 버튼을 누르면 작은 메뉴가 뜹니다.

- **빈 페이지** — 즉시 빈 BlockNote 페이지 생성
- **템플릿에서 시작** — 다음 카드 중 선택:
  - **빈 페이지** (템플릿 갤러리 안에서도 다시 선택 가능)
  - **프로젝트 \| 상태별** — 데이터베이스 페이지 (Board view, 칸반)
  - **일정 \| 마감일별** — 데이터베이스 페이지 (Calendar view)
- **HTML 페이지 (실험)** — HTML View 페이지를 만들고 자동으로 편집 모드로 진입

## 편집 (BlockNote)

본문 영역에 입력하면 됩니다. 모든 블록은 슬래시 명령(`/`) 으로 추가합니다.

지원 블록:

- 헤딩 (H1 / H2 / H3)
- 단락
- 글머리 / 번호 / 체크박스 목록
- 인용
- 코드 블록 (언어별 하이라이트)
- 표 (헤더 행 굵게 가능)
- 이미지 (드래그&드롭 또는 슬래시)
- 구분선 (divider)
- 멘션 (`@`)

자동 저장됩니다 — 입력 후 잠깐 쉬면 디스크에 반영됩니다.

## 페이지 속성

페이지 우측의 속성 패널 (또는 페이지 메뉴 안의 속성 항목).

| 속성 | 설명 |
|---|---|
| status | 할 일 / 진행 중 / 완료 등 (자유 텍스트) |
| doc_type | 문서 종류 분류 |
| tags | 다중 태그 |
| due_date | 마감일 |
| start_date | 시작일 (데이터베이스 캘린더 view 에서 활용) |
| priority | low / medium / high |
| assignee | 다중 사용자 또는 역할 |
| reviewer | 다중 사용자 또는 역할 |

assignee / reviewer 에 역할을 넣으면 그 역할 멤버가 자동으로 슬롯을 채웁니다. 자세한 내용은 [역할과 권한](/help/roles-permissions) 의 동적 그룹화 절을 참조하세요.

## 가시성 (공유 / 비공개)

사이드바의 페이지 항목 위에서 더보기 메뉴를 열어 가시성을 토글합니다.

- **공유(shared)** — 워크스페이스 멤버 전원 (`page.view` 권한이 필요)
- **비공개(private)** — 작성자 본인만

자식 페이지는 부모와 별개로 토글합니다. 한 번에 자식까지 묶어서 변경하는 옵션은 향후 추가 예정.

> 개인 워크스페이스에서는 의미가 없습니다 — 어차피 본인뿐이라 무엇으로 두든 본인만 봅니다.

## 멘션과 알림

본문에서 `@` 입력으로 워크스페이스 멤버 자동완성. 선택해서 멘션을 박으면 다음이 일어납니다.

- 상대방의 종 아이콘에 빨간 배지가 즉시 켜집니다 (다음 폴링 주기에)
- 상대방에게 이메일이 발송됩니다
- 같은 멘션을 다시 저장해도 재발송되지 않습니다 (save-time diff)
- 본인을 멘션해도 알림은 가지 않습니다

자세한 내용은 [알림](/help/notifications) 을 참조하세요.

## 공개 공유 링크

페이지 메뉴(페이지 상단의 더보기) 에서 "공유 링크" 항목을 엽니다.

`/share/<token>` 형식의 URL 이 발급됩니다. **이 링크를 가진 사람은 인증 없이 페이지를 볼 수 있습니다** (읽기 전용).

- 한 페이지에 활성 토큰 하나
- "새 토큰으로 회전" 버튼 — 기존 링크가 즉시 무효화됩니다
- "비활성화" — 공유 종료
- 페이지를 보관(archive) 하면 자동으로 비활성화됩니다

> HTML View 페이지는 공개 공유를 지원하지 않습니다 — 비-멤버에게 사용자 작성 JS 를 노출하면 위험하기 때문입니다.

## 인쇄와 PDF 저장

페이지 메뉴에서 "프린트 / PDF 저장" 을 선택합니다.

브라우저의 인쇄 대화 상자가 열립니다 — 거기서 "PDF로 저장" 을 고르면 됩니다.

자동으로 숨겨지는 것:

- 사이드바 / 워크스페이스 레일
- 토스트 알림
- 슬래시 메뉴 / BlockNote 툴바
- 이미지 다운로드 오버레이

> HTML View 페이지는 인쇄가 지원되지 않습니다 (sandbox iframe 의 제약).

## 보관함 (archive)

페이지 메뉴의 "보관함으로 이동" 을 사용합니다. 사이드바 트리에서 사라지지만 데이터는 그대로 유지됩니다.

- 사이드바 하단의 "보관함" 에서 복원 또는 영구 삭제
- 보관 시 공개 공유 링크는 자동 비활성화
- 다른 페이지에서 link 로 참조 중이면 끊긴 link 가 됩니다

## 페이지 트리

사이드바에서 페이지를 다른 페이지의 자식으로 드래그할 수 있습니다.

- 한 워크스페이스 안에서만 이동 — 워크스페이스 간 이동은 미지원
- 부모를 비공개로 바꿔도 자식의 가시성은 바뀌지 않습니다 (각각 독립)
- 자식이 있는 페이지를 보관하면 자식도 함께 보관됩니다

## 버전 충돌

같은 페이지를 두 사람이 거의 동시에 수정하면 어떻게 되는가:

- BlockNote 페이지 — 자동으로 Yjs 협업이 활성화되어 충돌이 자동 해결됩니다 ([실시간 협업](/help/realtime-collaboration))
- HTML View 페이지 — 협업 없음. 마지막 저장이 다른 사람 변경을 덮어씁니다. 충돌이 감지되면 새로고침 안내가 뜹니다
