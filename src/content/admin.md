# 시스템 어드민 가이드

hubNote 인스턴스 자체의 운영자(시스템 어드민) 가 사용하는 어드민 패널 안내입니다. 일반 워크스페이스 소유자와 혼동하지 않도록 주의하세요 — 자세한 차이는 [역할과 권한](/help/roles-permissions) 의 첫 절을 참조.

## 진입 조건

다음 두 가지가 모두 충족되어야 합니다.

1. 본인 계정에 어드민 권한이 부여되어 있을 것 (다른 어드민이 부여, 또는 인스턴스 초기 시드)
2. 현재 세션이 **AAL2** 일 것 — TOTP 를 등록했고 최근에 인증을 완료한 상태

조건이 안 맞으면 어드민 패널 진입 시:

- TOTP 미등록 — 등록 안내 화면이 뜹니다 (계정 설정의 보안 탭으로)
- AAL1 (TOTP 등록은 했지만 이번 세션에 인증 안 함) — TOTP 입력 화면이 뜹니다

## 어드민 패널 4탭

URL: `/admin/*`

### System

인스턴스 전반의 메트릭.

- 사용자 수 (총 / 활성 / 정지)
- DB 용량
- 파일 저장소 사용량
- 최근 24시간 / 7일 활동 요약

### Users

사용자 목록 / 검색 / 상세.

상세 페이지에서 가능한 액션:

| 액션 | 설명 |
|---|---|
| 관리자 부여 / 회수 | 어드민 권한 토글 |
| 정지 / 해제 | 정지된 사용자는 모든 API 가 403, 정지 안내 페이지로 이동 |
| 사용자명 쿨다운 리셋 | 30일 제한을 풀어줌 |
| 동의 bump | 약관 버전 강제 갱신 — 그 사용자는 다음 로그인 때 ConsentGate 재노출 |
| 강제 삭제 | 사용자 계정을 즉시 영구 삭제 (14일 soft-delete 우회) |

각 파괴적 액션은 매번 TOTP 재확인 (step-up) 을 요구합니다 — 세션이 AAL2 라도 별도 입력.

### Audit

모든 mutate 액션이 기록되는 감사 로그.

- 액터(누가) / 액션(무엇을) / 대상(누구에게) / IP / User-Agent / 시각
- 검색 가능 (사용자별 / 액션별 / 기간)
- 어드민 액션뿐 아니라 워크스페이스 / 페이지 변경도 기록됩니다

### Announcements

전사 공지 작성.

세 등급:

| 등급 | 표현 | 닫기 |
|---|---|---|
| info | 파란 배너 (사이드바 위) | 사용자가 X 클릭 시 그 사용자에게만 사라짐 |
| warning | 노란 배너 | 위와 동일 |
| critical | 빨간 배너 + **모달** | 사용자가 명시적으로 닫아야 사라짐 |

`/announcements` 에서 사용자가 전체 아카이브를 볼 수 있습니다. 닫은 공지는 그 사용자에게 다시 보이지 않습니다 (등급별 dismissal 추적).

## 자동 이메일 (어드민 액션 시)

다음 액션은 대상 사용자에게 자동으로 이메일이 발송됩니다.

- 관리자 부여 / 회수
- 계정 정지 / 해제
- 사용자명 쿨다운 리셋
- 동의 bump

본문은 액션별로 미리 정의되어 있으며, `RESEND_API_KEY` 가 설정되어 있어야 발송됩니다 (없으면 silent no-op, audit log 는 그대로 기록).

## Rate limit

`/api/admin/*` 의 mutate 엔드포인트는 분당 10회 / 토큰 당으로 제한됩니다 — 어드민 봇 또는 잘못된 클라이언트 폭주 방지.

초과 시 429 응답 + `Retry-After` 헤더. 보통 사람의 사용 패턴에서는 도달하지 않습니다.

## 운영 권장

1. **본인 외에 1명 이상의 어드민** — 부재 시 잠금 위험
2. **`ADMIN_REQUIRE_AAL2=true`** 환경 변수 활성화 — 백엔드도 강제 (현재 기본 false 라 운영자 본인이 TOTP 등록 후 켜야 함)
3. **공지는 critical 을 자제** — 자주 띄우면 사용자가 무시하게 됩니다. 정말 시스템 차단되는 상황 외엔 warning 으로
4. **사용자 정지보다 데이터 보존** — 의심 사용자 발생 시 즉시 삭제 대신 정지로 전환, 14일 soft-delete 패턴 활용

## 자주 묻는 점

### 어드민 권한을 처음 어떻게 가지나

인스턴스 초기 시드 — `.env` 또는 DB 직접 조작으로 첫 어드민을 1명 만든 후, 그 사람이 어드민 패널에서 다른 어드민을 부여합니다.

### 어드민 권한을 잃었을 때

다른 어드민이 다시 부여해주거나, 시스템 운영자가 DB 에 직접 접근해서 `users.is_admin = true` 로.

### TOTP 를 분실했을 때 어드민 진입

본인 계정의 복구 코드 사용 (계정 설정에서 등록 시 발급받은 8개 중 하나). 그것도 없으면 인프라 운영자에게 직접 문의 — DB 에서 TOTP factor 를 삭제해 처음부터 등록하는 절차.
