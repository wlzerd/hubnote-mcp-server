# 데이터베이스 페이지

데이터베이스는 페이지의 한 종류입니다. 일반 페이지처럼 사이드바 트리에 나타나지만, 본문 자리에 카드 보드 또는 캘린더가 보입니다.

## 만들기

사이드바의 + 버튼에서 "템플릿에서 시작" 을 누르면 데이터베이스 템플릿이 보입니다.

- **프로젝트 \| 상태별** — Board view (칸반)
- **일정 \| 마감일별** — Calendar view (월 / 주 토글)

빈 데이터베이스를 만들려면 템플릿 갤러리의 빈 카드를 사용하거나, 기존 데이터베이스를 복제해서 비워주는 방법을 쓰세요. (사이드바 + 메뉴에 "데이터베이스" 직접 진입점은 향후 추가 예정.)

## View 종류

한 데이터베이스에 여러 view 를 만들 수 있습니다. 페이지 상단의 ViewBar 에서 전환합니다.

### Table

표 형식. 각 행이 한 카드(페이지) 입니다.

### Board (Kanban)

`status` 속성을 칼럼으로 사용. 카드를 다른 칼럼으로 드래그하면 그 페이지의 status 가 자동으로 갱신됩니다.

- 칼럼별로 빈 슬롯에 + 클릭으로 카드 추가
- 카드 클릭 시 속성 모달이 뜹니다 — 그 자리에서 편집 / 페이지로 이동 / 취소

### Calendar

`due_date` 속성을 기반으로 막대를 표시. month / week 토글이 있습니다.

- 다일(multi-day) 일정은 한 줄짜리 막대로 cell 을 가로지릅니다
- 주 경계를 넘는 일정은 각 행 끝에 화살표 표시
- 막대를 잡고 다른 날로 드래그하면 평행 이동합니다 (막대 길이는 유지)
- 마감일이 없는 카드는 별도 트레이로 분리

## 카드 = 페이지 (linked-database 시맨틱)

이 부분이 노션과 다르게 헷갈릴 수 있는 점입니다.

데이터베이스 카드는 **페이지 자체** 를 가리킵니다. 카드는 데이터베이스에 직접 데이터를 박는 게 아니라 페이지로 가는 link 입니다.

### 카드 추가 시

두 가지 선택:

- **기존 페이지 link 걸기** — 이미 만들어둔 페이지를 그 데이터베이스의 멤버로 추가
- **새 페이지 만들면서 link** — 페이지를 새로 만들고 자동으로 그 데이터베이스에 link

### 같은 페이지가 여러 보드에 동시 등장

한 페이지가 여러 데이터베이스에 동시에 등록될 수 있습니다.

- 예: "디자인 시안 검토" 페이지가 "프로젝트 보드" 와 "검토 캘린더" 양쪽에 카드로 표시
- 한쪽에서 status 를 바꾸면 페이지 본체의 속성이 바뀌니 양쪽 카드에 반영됩니다
- 페이지 본체의 위치(parent_id) 와 데이터베이스 멤버십은 무관합니다

### 카드를 데이터베이스에서 빼기

카드 메뉴에서 "이 데이터베이스에서 제거" — 페이지 자체는 사라지지 않고, 그 데이터베이스의 카드 link 만 제거됩니다.

## 카드 클릭 동작

카드 본체를 클릭하면 작은 모달이 뜹니다.

- **속성 편집** — 모달 안에서 status / 마감일 / 담당자 등을 직접 수정. 페이지 본문은 안 열림
- **문서로 이동** — 페이지 본문 화면으로 전환
- **취소** — 모달 닫기

## 권한

데이터베이스 view 와 row 에 대한 권한은 별도 슬러그로 관리됩니다 ([역할과 권한](/help/roles-permissions) 참조).

- `database.manage_views` — view 추가 / 삭제 / 설정
- `database.create_rows` — 카드 추가
- `database.edit_rows` — 카드 속성 수정 + 드래그&드롭

페이지 본체 편집은 `page.edit` 가 따로 필요합니다 — 데이터베이스 권한과 페이지 권한이 분리되어 있습니다.

## 워크스페이스 데이터 row (별도 시스템)

페이지 기반 데이터베이스 외에, 워크스페이스 단위 자유 형식 데이터 row 시스템이 따로 있습니다.

- HTML View 페이지가 SDK 로 다루는 그 데이터입니다
- 페이지 트리에 보이지 않으며, HTML View 또는 외부 도구에서만 다룹니다
- 자세한 사용법은 [HTML View 개발자 가이드](/help/html-view-developer)

## 알려진 한계 (1차 스코프)

- 같은 칼럼 / 같은 날짜 안에서 카드 순서 바꾸기 미지원 (다른 칼럼 / 날짜 이동만)
- 캘린더 막대 좌우 끝을 잡고 늘려서 기간 변경 미지원 (속성 모달에서 직접 날짜 편집)
- 각 view 별 필터 / 정렬 UI 미지원 (백엔드는 일부 지원, UI 미연결)
- 사이드바 + 메뉴에 "데이터베이스" 직접 진입점 미지원 (템플릿 갤러리 경유)
