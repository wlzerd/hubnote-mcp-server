# 역할과 권한

워크스페이스 멤버에게 무엇을 할 수 있는지 부여하는 시스템입니다.

## 기본 개념

- **역할(Role)** — 권한의 묶음. 워크스페이스마다 여러 개를 만들 수 있습니다
- **권한 슬러그(Permission slug)** — 한 가지 작업의 허용 여부 (총 15개)
- **멤버 ↔ 역할은 다대다** — 한 멤버에게 여러 역할을 동시에 부여할 수 있고, 권한은 합집합으로 적용됩니다

## 워크스페이스 소유자 vs 시스템 어드민 (혼동 주의)

두 단어가 한국어로 다 "관리자" 로 번역되기 쉬워서 헷갈립니다.

- **워크스페이스 소유자(owner)** — 그 워크스페이스를 만든 사람. 자기 워크스페이스에 한해 모든 권한이 자동 통과됩니다. 다른 워크스페이스에서는 일반 멤버
- **시스템 어드민** — hubNote 인스턴스 자체의 운영자. 별도 부여가 필요하며 모든 워크스페이스에 영향을 미칠 수 있습니다. 자세한 내용은 [시스템 어드민 가이드](/help/admin)

이 문서에서 다루는 "관리자" 는 모두 **워크스페이스 소유자 또는 관리자 역할이 부여된 멤버** 를 가리킵니다.

## 5가지 프리셋

워크스페이스를 만들면 다음 프리셋이 함께 생성됩니다. 모두 끄거나 커스터마이즈 가능합니다.

| 프리셋 | 권한 |
|---|---|
| 관리자 | 모든 권한 |
| 편집자 | 페이지 / 데이터베이스 작성·편집·삭제, HTML View 작성·편집 |
| 검토자 | 페이지 보기 + 댓글 (댓글은 향후 활성화) |
| 콘텐츠 작성자 | 일반 페이지 / HTML View 만 작성·편집 |
| 데이터 작성자 | 데이터베이스 row 만 작성·편집 |

## 권한 슬러그 (총 15개)

5개 그룹으로 분류됩니다.

### 워크스페이스 (4)

- `workspace.manage_members` — 멤버 추가 / 제거
- `workspace.manage_settings` — 이름·이미지·설정 변경
- `workspace.manage_roles` — 역할 만들기 / 권한 부여
- `workspace.manage_invites` — 초대 링크 발급 / 회전 / 비활성화

### 페이지 (5)

- `page.view` — 공유 페이지 보기
- `page.create` — 페이지 만들기
- `page.edit` — 다른 사람의 페이지 편집
- `page.delete` — 페이지 보관함 이동 / 영구 삭제
- `page.manage_visibility` — 페이지 가시성 (공유 ↔ 비공개) 변경

### HTML View (2)

- `html_view.create` — HTML View 페이지 작성
- `html_view.edit` — 다른 사람의 HTML View 편집

### 데이터베이스 (3)

- `database.manage_views` — view 추가 / 삭제 / 설정
- `database.create_rows` — row (카드) 추가
- `database.edit_rows` — row 편집

### 파일 (1)

- `file.upload` — 본문에 파일 / 이미지 업로드

## 자주 헷갈리는 점

- **소유자는 자동 통과합니다.** 권한 토글이 비어있어도 막히지 않습니다
- **페이지 작성자는 자기 페이지에 자동 통과합니다.** 다른 사람이 만든 페이지는 권한 슬러그가 필요합니다
- 권한 없는 작업 시도 시 한국어로 "워크스페이스 관리자에게 문의" 토스트가 뜹니다
- 새 권한 슬러그가 도입되면 기본은 OFF — 기존 멤버에게 자동 부여되지 않으니 관리자가 명시적으로 켜야 합니다

## 동적 그룹화 (역할 기반 collaborator)

페이지의 담당자(assignee) 와 검토자(reviewer) 슬롯에는 사용자뿐 아니라 **역할** 도 지정할 수 있습니다.

- 예: "팀장 역할이 검토자" — 팀장 역할을 가진 모든 멤버가 자동으로 그 페이지의 검토자가 됩니다
- 역할 멤버가 바뀌면 페이지 속성도 자동으로 따라갑니다 (페이지를 매번 수정할 필요 없음)
- 한 페이지에 여러 명 / 여러 역할을 동시 지정 가능

## 역할 만들기와 편집

워크스페이스 설정의 역할 탭에서 작업합니다.

- "+ 새 역할" 으로 빈 역할 만들기
- 카드를 펼쳐서 권한 슬러그를 토글
- 5개 프리셋 버튼으로 빠른 채우기
- "편집 모드 진입" 시에만 권한 목록이 펼쳐집니다 — 평소엔 카드만 보여 깔끔합니다

## 운영 권장 (소유자용)

1. **관리자 역할을 만들어 본인 + 부 운영자에게 부여하세요** — 소유자 1명에 의존하면 부재 시 곤란해집니다
2. **편집자 / 검토자 / 데이터 작성자** 같은 프리셋 중 멤버 성격에 맞게 부여
3. **민감한 페이지는 비공개(private)** 로 — 본인만 보입니다
4. **HTML View 페이지를 멤버에게 위임할 때**: 그 사람을 페이지 작성자로 두면 본인 + 소유자 둘 다 편집 가능. 작성자가 워크스페이스를 떠나도 소유자는 계속 편집 가능
