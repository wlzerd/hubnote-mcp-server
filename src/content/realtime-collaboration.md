# 실시간 협업

같은 페이지를 여러 명이 동시에 편집할 수 있습니다. 충돌은 자동으로 해결됩니다.

## 자동 활성화

별도 설정이 필요 없습니다. BlockNote 페이지를 열면 자동으로 협업이 켜집니다.

- 다른 사람이 같은 페이지에 들어오면 본인 화면에 그 사람의 커서가 보입니다
- 동시 입력은 Yjs CRDT 가 자동으로 머지합니다 (마지막에 저장한 사람이 이기는 게 아닙니다)
- 잠깐 끊겼다가 다시 연결되어도 그동안의 변경은 유실되지 않고 머지됩니다

## 어떤 페이지에서 작동하나

| 페이지 종류 | 협업 |
|---|---|
| 일반 BlockNote 페이지 | 자동 활성화 |
| 데이터베이스 페이지 | view 자체는 단일 작성, 카드(페이지) 본문은 BlockNote 라 협업 |
| HTML View 페이지 | 미지원 — 단일 작성자, optimistic lock |

## 커서 라벨

다른 사람이 본인의 본문에 들어오면 그 사람의 커서가 색깔 막대 + 라벨로 표시됩니다.

- 라벨에 표시 이름 (display_name)
- 아바타 사진을 등록한 사람은 라벨 안에 작은 동그라미로 함께 보입니다
- 색깔은 사용자 ID 에 기반한 안정적인 해시 — 같은 사람은 항상 같은 색
- 아바타 로딩이 실패하면 라벨에 사진은 빠지고 이름만 보입니다

## 알려진 한계

### 첫 오픈 시드 race

완전히 비어있는 새 페이지를 두 사람이 거의 같은 순간에 처음 열면, 드물게 본문이 중복으로 시드될 수 있습니다.

- 페이지 본문에 같은 내용이 두 번 들어가는 형태로 나타납니다
- 데이터 손실은 없으며, 한쪽을 지우면 됩니다
- 한 사람이 먼저 열고 잠깐 입력한 후 다른 사람이 들어오면 발생하지 않습니다

### 검색 인덱스 lag

검색 인덱스는 본인이 마지막으로 직접 저장한 시점 기준으로 갱신됩니다.

- 다른 사람만 편집한 변경은 다음에 본인이 그 페이지를 열어 저장할 때 인덱스에 반영됩니다
- 짧은 기간 동안 검색 결과가 약간 오래된 텍스트로 노출될 수 있습니다 — 향후 개선 예정

### HTML View 는 협업 미지원

[HTML View](/help/html-view) 페이지는 단일 작성자입니다. 두 사람이 동시에 저장을 시도하면 마지막 저장이 다른 사람 변경을 덮어씁니다 (충돌 시 새로고침 안내가 뜸).
