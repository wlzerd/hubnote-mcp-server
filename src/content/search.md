# 검색

hubNote 의 모든 페이지를 한 번에 검색합니다. 한국어 풀텍스트가 기본으로 활성화되어 있습니다.

## 검색 팔레트 열기

- **Cmd + K** (Mac) / **Ctrl + K** (Windows) — 어디서든 즉시 열림
- 사이드바의 "검색" 행 클릭으로도 열 수 있습니다

## 검색 대상

- 페이지 제목 (자동 인덱싱)
- 페이지 본문 (저장 시 자동 인덱싱)
- 멘션 본문 (`@사용자명` 의 표시 이름까지 포함)

본인이 볼 수 있는 페이지만 결과에 노출됩니다 — 비공개 페이지는 작성자에게만, 공유 페이지는 워크스페이스 멤버에게만.

다음 페이지 종류는 검색 인덱스에서 제외됩니다.

- HTML View — 본문이 코드라 검색 의미 없음
- 보관함의 페이지 — 인덱스에서 즉시 제거됨

## 워크스페이스 스코프

팔레트 안의 토글로 검색 범위를 전환할 수 있습니다.

- **현재 워크스페이스** (기본) — 활성 워크스페이스만
- **전체 워크스페이스** — 본인이 속한 모든 워크스페이스

## 검색 연산자

PGroonga 가 지원하는 문법입니다.

| 연산자 | 의미 | 예시 |
|---|---|---|
| 공백 | AND | `프로젝트 회의` — 두 단어 모두 포함 |
| `OR` | OR | `프로젝트 OR 회의` — 둘 중 하나 |
| `-단어` | 제외 | `프로젝트 -완료` — "완료" 제외 |
| `"구문"` | 정확한 구문 | `"분기 보고서"` |

## 한국어 검색 팁

PGroonga 는 한국어 형태소 분석을 합니다.

- 어미 변화: "회의" 검색 시 "회의가" / "회의를" 도 매칭
- 영어 / 한국어 혼합 검색 가능
- 따옴표를 빈 채로 끝나는 미완성 입력은 자동으로 무시됩니다 (검색이 매번 400 으로 깨지는 걸 방지)

## 결과 화면

각 항목에 다음이 표시됩니다.

- 페이지 제목 (매칭 부분 강조)
- 본문 일부 (매칭 부분 주변 발췌)
- 워크스페이스 이름 (전체 스코프일 때)
- 페이지 종류 (database / 일반 / 보관함 등)

결과를 클릭하면 그 페이지로 이동합니다.

## 알려진 한계

- 협업 페이지에서 다른 사람만 편집한 변경은 다음에 본인이 저장할 때까지 인덱스에 반영되지 않습니다 ([실시간 협업](/help/realtime-collaboration) 의 검색 인덱스 lag 절 참조)
- 인덱스 재구축은 자동이지만, 마이그레이션 직후나 대규모 백필 후엔 잠시 누락이 있을 수 있습니다 — 운영자가 백필 스크립트로 보정합니다
