# 파일

페이지 본문에 첨부하는 이미지와 일반 파일에 관한 안내입니다.

## 업로드

본문 편집 중에 다음 방법으로 업로드합니다.

- 본문에 파일을 드래그&드롭
- 슬래시 메뉴(`/`) 의 "이미지" / "파일" 항목
- HTML View 에서는 `ws.uploadFile(file)` SDK 호출

## 사용자별 쿼터

- 누적 5GiB
- 일일 업로드 횟수 제한 (운영자 측 환경 변수로 조정)
- MIME 타입 화이트리스트 — 허용된 종류 외 거부 (이미지 / 일반 문서 / 텍스트)

쿼터 초과 시 업로드 거부 메시지가 뜹니다. 사용량은 계정 설정의 "내 파일" 탭에서 확인할 수 있습니다.

## 파일 URL

업로드된 파일은 `/files/<uuid>` 형식의 URL 로 접근합니다.

- UUID 는 추측 불가능한 랜덤
- 인증 없이 URL 만으로 접근 가능 — 노션과 같은 정책
- URL 이 외부에 새 나가면 그 파일은 누구나 볼 수 있다는 점 유의

> 민감한 파일은 페이지 본문에 직접 박지 말고, 별도 도구로 보관하세요. URL 추측 불가 + 본문 권한 + 토큰 회전이 함께 작동하는 구조라 일반적인 사용엔 안전합니다.

## 내 파일 탭 (계정 설정)

계정 설정 → "내 파일" 탭에서:

- 누적 사용량 / 5GiB 진행 바
- 일일 업로드 카운터
- 본인이 업로드한 파일 목록 (최대 200개)
- 행별 삭제 버튼

### 삭제 정책

- 본인이 업로드한 파일만 삭제 가능 (다른 사람의 업로드는 보이지도 않음)
- 삭제 시 DB row 가 먼저 사라지고, 디스크의 실제 파일은 best-effort 로 unlink 됩니다
- 페이지 본문에서 그 파일을 link 하고 있어도 자동으로 그 link 가 사라지지 않습니다 — 일반 404 가 됩니다

## 이미지 다운로드

페이지 본문의 이미지 위에 마우스를 올리면 우상단에 다운로드 버튼이 뜹니다. 클릭하면 원본 파일로 저장됩니다.

## HTML View 에서의 파일

[HTML View 페이지](/help/html-view) 안에서는 두 방법으로 이미지를 다룹니다.

- 외부 https URL — 자동으로 백엔드 image-proxy 경유
- 자체 업로드 — `ws.uploadFile(file)` 호출 후 반환된 `/files/<uuid>` 사용

자세한 내용은 [HTML View 개발자 가이드](/help/html-view-developer) 의 이미지 처리 절을 보세요.

## 파일이 사라졌을 때

- 본인이 삭제 — "내 파일" 에서 의도적으로 삭제했을 가능성
- 다른 사람이 같은 파일 link 를 가지고 있던 경우 — 그쪽 본문의 이미지가 깨집니다
- 운영자 쪽 디스크 사고 — 백업으로 복원 (운영서버는 매일 `pg_dump` + 향후 객체 스토리지 마이그레이션 예정)

## 자주 묻는 점

### 비디오 / 오디오 업로드는

현재 MIME 화이트리스트는 이미지와 일반 문서 중심입니다. 큰 미디어 파일은 외부 호스팅 (YouTube 등) 에서 가져오는 게 권장됩니다.

### 파일 이름이 한글인데 깨지나

업로드 시 원본 파일명은 메타로 저장되고, 다운로드 시에도 한글 그대로 복원됩니다. 일부 구버전 브라우저에서는 다운로드 다이얼로그의 파일명이 변환될 수 있습니다.
